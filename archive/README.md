# 归档文件说明

本目录用于归档项目中的进度报告、状态更新、完成总结等与核心主题无关的文件。

## 目录结构

```
archive/
├── progress-reports/          # 根目录进度报告
│   ├── 2024/                  # 2024年报告
│   └── 2025/                  # 2025年报告
├── concepts-reports/          # Concepts模块报告
│   ├── phase1/                # Phase1相关报告
│   └── cross-ref/             # 交叉引用相关报告
├── philosophy-reports/        # Philosophy模块报告
│   ├── execution/             # 执行相关报告
│   ├── improvement/           # 改进相关报告
│   └── completion/            # 完成相关报告
└── docs-reports/              # Docs模块报告
    └── completion/            # 完成相关报告
```

## 归档原则

1. **进度报告文件**：所有PROGRESS、STATUS、UPDATE、COMPLETION、SUMMARY等类型的文件
2. **交叉引用报告**：所有cross_ref、cross-reference相关的报告文件
3. **改进工作报告**：所有IMPROVEMENT、REPAIR相关的报告文件
4. **执行跟踪文件**：所有EXECUTION、TRACKER相关的文件
