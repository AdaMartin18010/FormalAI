## 1. 计算机视觉应用

### 1.1 图像分类与目标检测

#### 问题定义

图像分类是将图像分配到预定义类别的任务，目标检测则需要在图像中定位并识别多个目标对象。这是计算机视觉最基础且应用最广泛的任务。

#### 技术方案演进

| 模型系列 | 代表模型 | 核心特点 | 适用场景 |
|---------|---------|---------|---------|
| YOLO系列 | YOLOv5/v8/v10 | 单阶段检测，实时性强 | 实时检测、边缘部署 |
| DETR系列 | DETR, Deformable DETR | 端到端Transformer架构 | 复杂场景、长距离依赖 |
| 两阶段检测 | Faster R-CNN, Mask R-CNN | 精度高，速度较慢 | 高精度要求场景 |
| 分割模型 | SAM, SAM 2 | 零样本分割能力 | 交互式分割、医学影像 |

#### YOLO系列深度解析

**YOLOv8架构特点：**

```
输入图像 → Backbone(CSPDarknet) → Neck(PANet) → Head(Decoupled Head) → 输出
                    ↓
              特征金字塔网络(FPN)
```

**核心创新：**

- **解耦头(Decoupled Head)**：将分类和回归任务分离，提升收敛速度
- **Anchor-Free设计**：直接预测目标中心点和尺寸，简化训练流程
- **CIOU Loss**：更精确的边界框回归损失函数

**性能对比（COCO数据集）：**

| 模型 | 参数量 | mAP@50-95 | 推理速度 (FPS) |
|-----|-------|----------|---------------|
| YOLOv5n | 1.9M | 28.0% | 1200+ |
| YOLOv8n | 3.2M | 37.3% | 800+ |
| YOLOv8s | 11.2M | 44.9% | 500+ |
| YOLOv8m | 25.9M | 50.2% | 300+ |
| YOLOv10 | - | 52.8% | 600+ |

#### DETR与Transformer检测

**DETR (Detection Transformer) 架构：**

```
CNN Backbone → Transformer Encoder → Transformer Decoder → FFN → 预测
                    ↓                      ↓
               全局特征提取          Object Queries
```

**核心优势：**

- 端到端训练，无需NMS后处理
- 全局上下文建模能力强
- 对遮挡和小目标检测更鲁棒

**Deformable DETR改进：**

- 引入可变形注意力，降低计算复杂度
- 多尺度特征融合，提升小目标检测性能
- 训练收敛速度提升10倍

#### SAM (Segment Anything Model)

**架构设计：**

```
图像编码器(ViT) → 提示编码器 → 轻量级掩码解码器 → 分割掩码
      ↓               ↓
   图像嵌入      点/框/文本提示
```

**应用场景：**

- 交互式图像编辑
- 医学影像分割
- 自动驾驶场景理解
- 增强现实内容生成

```
          ▼                ▼
```

| 实时性要求(<200ms) | 模型量化(INT8) + 批处理推理 |

### 1.2 图像分割

#### 分割类型对比

| 分割类型 | 定义 | 输出 | 典型应用 |
|---------|-----|-----|---------|
| 语义分割 | 像素级类别分类 | 类别掩码 | 场景理解、医学影像 |
| 实例分割 | 区分不同实例 | 实例掩码 | 自动驾驶、机器人 |
| 全景分割 | 语义+实例统一 | 统一掩码 | 街景分析、AR/VR |

#### 主流分割模型

**U-Net系列（医学影像首选）：**

```
编码器: 卷积下采样 → 提取多尺度特征
           ↓
      跳跃连接 (Skip Connection)
           ↓
解码器: 上采样 → 精确定位分割边界
```

**DeepLab系列：**

- **ASPP模块**：多尺度空洞卷积捕获上下文
- **DeepLabV3+**：编码器-解码器结构，边界更精细

**Mask R-CNN：**

```
Faster R-CNN → ROI Align → 并行分支
                              ↓
                    ┌────────┴────────┐
                    ↓                 ↓
                 分类+边界框        掩码预测
```

#### 医学影像分析深度案例

**肺癌早期筛查系统：**

**数据流程：**

```
CT扫描(3D体积) → 肺部分割 → 结节检测 → 良恶性分类 → 报告生成
     ↓              ↓          ↓           ↓
  DICOM格式    U-Net 3D   3D CNN    多任务学习模型
```

**技术细节：**

- **预处理**：窗位窗宽调整(-600HU肺窗)、重采样到统一体素间距
- **结节检测**：3D U-Net + 候选区域筛选
- **假阳性降低**：多视角CNN集成
- **良恶性预测**：LUNA16竞赛方案 + 临床特征融合

**性能指标：**

- 结节检测灵敏度：95%@4 FPs/scan
- 良恶性分类AUC：0.92
- 推理时间：<30秒/CT扫描

### 1.3 人脸识别与生物特征

#### 技术架构

## 3. 推荐系统

### 3.1 协同过滤与矩阵分解

#### 问题定义

根据用户历史行为和物品特征，预测用户对未交互物品的偏好，实现个性化推荐。

#### 协同过滤类型

| 类型 | 原理 | 优点 | 缺点 |
|-----|-----|-----|-----|
| 用户协同过滤 | 找相似用户 | 可解释性强 | 用户稀疏性 |
| 物品协同过滤 | 找相似物品 | 物品稳定性高 | 冷启动问题 |
| 矩阵分解 | 隐向量学习 | 泛化能力强 | 缺乏可解释性 |

#### 矩阵分解技术

**SVD (Singular Value Decomposition)：**

```
R ≈ U × Σ × V^T

R: 用户-物品评分矩阵 (m×n)
U: 用户隐向量矩阵 (m×k)
Σ: 奇异值对角矩阵 (k×k)
V: 物品隐向量矩阵 (n×k)
```

**ALS (Alternating Least Squares)：**

```python
# 交替优化用户和物品向量
while not converged:
    # 固定物品向量，优化用户向量
    for u in users:
        X_u = (Y^T * Y + λI)^(-1) * Y^T * r_u

    # 固定用户向量，优化物品向量
    for i in items:
        Y_i = (X^T * X + λI)^(-1) * X^T * r_i
```

**NMF (Non-negative Matrix Factorization)：**

- 约束隐向量非负
- 更符合直观解释（用户偏好、物品属性）

### 3.2 深度推荐模型

#### 模型演进路线

```
传统ML → 浅层NN → 深度模型 → 序列模型 → 图神经网络 → 大模型
  ↓         ↓         ↓          ↓          ↓          ↓
MF/CF   Wide&Deep   DeepFM    DIN/DIEN   NGCF/PinSage   LLM4Rec
```

#### Wide & Deep

**架构设计：**

```
输入特征
    │
    ├──→ Wide部分 (线性模型) ──┐
    │                           ├──→ 联合训练 → 输出
    └──→ Deep部分 (DNN) ───────┘
              ↓
         Embedding + 多层MLP
```

**核心思想：**

- **Wide**：记忆能力，学习特征交叉规则
- **Deep**：泛化能力，学习隐式特征表示
- **联合训练**：平衡记忆与泛化

**适用场景：**

- Google Play应用推荐
- 特征工程较完善的场景

#### DeepFM

**架构设计：**

```
输入特征
    │
    ├──→ FM部分 (二阶交叉) ──┐
    │                         ├──→ Sigmoid → 输出
    └──→ Deep部分 (DNN) ─────┘
```

**FM (Factorization Machines)：**

```
y = w_0 + Σ w_i x_i + Σ Σ <v_i, v_j> x_i x_j
                    i j>i

<v_i, v_j>: 隐向量内积，自动学习特征交叉
```

**优势：**

- 端到端训练，无需人工特征工程
- 低阶+高阶特征联合建模
- 计算效率高

#### DIN (Deep Interest Network)

**核心创新 - 注意力机制：**

```
用户历史行为物品: [item_1, item_2, ..., item_n]
候选物品: item_target

注意力权重: w_i = f(item_i, item_target)

用户兴趣向量: Σ w_i * item_i
```

**架构特点：**

- 引入注意力机制建模用户多样兴趣
- 不同候选物品激活不同历史行为
- 解决传统方法兴趣向量固定的问题

#### DIEN (Deep Interest Evolution Network)

**核心创新：**

```
用户行为序列 → GRU → 注意力GRU (AUGRU) → 兴趣演化表示
                    ↓
               候选物品引导
```

**兴趣演化建模：**

- 捕捉用户兴趣随时间变化
- 注意力机制筛选相关历史行为
- 更适合序列推荐场景

#### 深度推荐模型对比

| 模型 | 核心创新 | 适用场景 | 计算复杂度 |
|-----|---------|---------|----------|
| Wide&Deep | 联合训练 | 通用场景 | 低 |
| DeepFM | 端到端FM | 特征交叉重要 | 中 |
| DCN | 显式高阶交叉 | 特征工程需求高 | 中 |
| xDeepFM | CIN压缩交互 | 高阶特征重要 | 高 |
| DIN | 注意力兴趣 | 行为序列丰富 | 中 |
| DIEN | 兴趣演化 | 时序推荐 | 高 |
| BST | Transformer序列 | 长序列建模 | 高 |

### 3.3 序列推荐

#### 问题定义

| 方法 | 代表模型 | 特点 | 适用场景 |

**核心设计：**

- 自注意力捕捉序列内依赖
- 位置编码保留时序信息
- 支持长序列建模（512+）

**架构：**


**优势：**

- 双向上下文建模
- 预训练+微调范式
- 序列理解能力强

## 4. 时序预测

### 4.1 传统时序模型

#### 问题定义

根据历史时间序列数据，预测未来时间点的数值。

#### ARIMA模型

**模型结构：**

```
ARIMA(p, d, q) = 差分(d) + AR(p) + MA(q)

AR(p): 自回归项，用过去p期值回归
MA(q): 移动平均项，用过去q期预测误差修正
```

**适用场景：**

- 数据量较小（<1000点）
- 趋势和季节性明显
- 需要可解释性

**参数选择：**

```python
# ACF/PACF分析确定p, q
# AIC/BIC准则选择最优参数
from statsmodels.tsa.arima.model import ARIMA

model = ARIMA(series, order=(p, d, q))
model_fit = model.fit()
forecast = model_fit.forecast(steps=10)
```

#### Prophet (Facebook)

**模型分解：**

```
y(t) = g(t) + s(t) + h(t) + ε_t

g(t): 趋势项 (分段线性/逻辑增长)
s(t): 季节性 (傅里叶级数)
h(t): 节假日效应
ε_t: 噪声项
```

**核心优势：**

- 自动处理缺失值和异常值
- 内置节假日效应
- 可解释性强
- 无需特征工程

**适用场景：**

- 业务数据预测（销售、DAU）
- 有明显季节性和节假日效应
- 需要快速建模

```python
from prophet import Prophet

model = Prophet(
    yearly_seasonality=True,
    weekly_seasonality=True,
    daily_seasonality=False,
    holidays=holiday_df
)
model.add_country_holidays(country_name='CN')
model.fit(df)
future = model.make_future_dataframe(periods=30)
forecast = model.predict(future)
```

### 4.2 深度学习时序模型

#### 模型演进

```
RNN/LSTM → Seq2Seq → Attention → Transformer → 专用架构
   ↓          ↓          ↓           ↓            ↓
基础序列    多步预测   长距离依赖   并行计算     时序特性
```

#### LSTM/GRU

**LSTM单元结构：**

```
遗忘门: f_t = σ(W_f · [h_{t-1}, x_t] + b_f)
输入门: i_t = σ(W_i · [h_{t-1}, x_t] + b_i)
候选状态: C̃_t = tanh(W_C · [h_{t-1}, x_t] + b_C)
细胞状态: C_t = f_t * C_{t-1} + i_t * C̃_t
输出门: o_t = σ(W_o · [h_{t-1}, x_t] + b_o)
隐藏状态: h_t = o_t * tanh(C_t)
```

**适用场景：**

- 中等长度序列（<500）
- 需要捕捉时序依赖
- 计算资源有限

#### Transformer时序模型

**Informer（长序列预测）：**

```
ProbSparse Attention: 选择重要Query，降低复杂度O(L log L)
Self-attention Distilling: 下采样，突出主导注意力
Generative Decoder: 一次前向传播生成长序列
```

**Autoformer（自相关机制）：**

```
序列分解: 趋势 + 季节性
自相关机制: 基于周期性的时间延迟聚合
复杂度: O(L log L)
```

#### N-BEATS/N-BEATSx

**架构设计：**

```
输入 → 全连接块堆叠 → 基函数展开 → 预测
         ↓
    残差连接
```

**核心创新：**

- 纯全连接架构，无RNN/CNN
- 可解释性分解（趋势+季节性）
- 堆叠结构，逐层优化

**N-BEATSx扩展：**

- 引入外生变量
- 适合多变量时序预测

#### PatchTST

**架构设计：**

```
时序序列 → 分块(Patch) → 线性嵌入 → Transformer Encoder → 预测
                                              ↓
                                        通道独立性
```

**核心优势：**

- 通道独立：每个变量独立处理
- Patch设计：保留局部语义
- 长期预测SOTA性能

#### 深度学习时序模型对比

| 模型 | 核心创新 | 最大序列长度 | 适用场景 |
|-----|---------|-------------|---------|
| LSTM | 门控机制 | 500 | 中等序列 |
| Informer | ProbSparse Attention | 5000 | 长序列预测 |
| Autoformer | 自相关机制 | 5000 | 周期性数据 |
| N-BEATS | 纯MLP+可解释 | 1000 | 单变量预测 |
| PatchTST | Patch+通道独立 | 10000 | 多元长序列 |
| TimesNet | 2D时序建模 | 10000 | 复杂模式 |

### 5.2 图像生成

#### 技术演进

```
GAN (2014) → VQ-VAE → DALL-E → Diffusion → Stable Diffusion → DALL-E 3
  ↓            ↓        ↓          ↓              ↓              ↓
对抗训练    向量量化   CLIP对齐   去噪过程      潜在空间        指令遵循
```

#### 扩散模型原理

**前向扩散（加噪）：**

```
x_t = √(α_t) * x_0 + √(1-α_t) * ε

x_0: 原始图像
x_t: t时刻的噪声图像
ε: 高斯噪声
α_t: 预设的噪声调度
```

**反向去噪（生成）：**

```
训练目标：预测噪声 ε_θ(x_t, t)

损失函数：L = ||ε - ε_θ(x_t, t)||²

生成过程：从纯噪声逐步去噪得到图像
```

#### Stable Diffusion架构

```
文本提示 → CLIP文本编码器 → 文本嵌入
                                  ↓
随机噪声 → U-Net (去噪) ← 条件注入 → 潜在空间图像
              ↓
         VAE解码器
              ↓
         生成图像
```

**核心组件：**

- **VAE**：图像与潜在空间转换
- **U-Net**：噪声预测网络
- **CLIP**：文本-图像对齐
- **Scheduler**：去噪调度策略

### 6.1 异常检测与欺诈检测

#### 异常检测类型

| 类型 | 定义 | 应用场景 |
|-----|-----|---------|
| 点异常 | 单个数据点异常 | 交易欺诈 |
| 上下文异常 | 特定上下文下异常 | 时间序列异常 |
| 集合异常 | 数据子集异常 | 团伙欺诈 |

#### 技术方案

| 方法 | 算法 | 特点 | 适用场景 |
|-----|-----|-----|---------|
| 统计方法 | 3σ, IQR, Z-score | 简单可解释 | 单变量、正态分布 |
| 聚类方法 | DBSCAN, Isolation Forest | 无监督 | 无标签数据 |
| 深度方法 | Autoencoder, VAE | 复杂模式 | 高维数据 |
| 图方法 | GNN异常检测 | 关系异常 | 网络数据 |

#### Isolation Forest

**原理：**

```
异常点更容易被孤立（更短的平均路径长度）

构建多棵随机二叉树，计算样本的平均路径长度
路径长度短 → 异常概率高
```

**优点：**

- 线性时间复杂度
- 无需假设数据分布
- 对高维数据有效

#### 深度异常检测

**Autoencoder：**

```
输入 → 编码器 → 低维表示 → 解码器 → 重建
                              ↓
                         重建误差大 → 异常
```

**VAE (Variational Autoencoder)：**

- 学习数据分布
- 重建概率作为异常分数
- 更适合复杂分布

#### 欺诈检测深度案例：金融反欺诈系统

## 8. 技术选型指南

### 8.1 场景-技术匹配矩阵

| 应用场景 | 推荐技术 | 备选方案 | 关键考虑因素 |
|---------|---------|---------|------------|
| 图像分类 | ResNet/EfficientNet | ViT, ConvNeXt | 精度vs速度 |
| 目标检测 | YOLOv8 | DETR, RT-DETR | 实时性要求 |
| 图像分割 | SAM | U-Net, DeepLab | 交互性需求 |
| 文本分类 | BERT/RoBERTa | GPT-4微调 | 数据量 |
| 文本生成 | GPT-4/Claude | LLaMA, Qwen | 成本vs质量 |
| 推荐系统 | DeepFM/DIN | Two-Tower, DSSM | 实时性 |
| 时序预测 | PatchTST | N-BEATS, Prophet | 序列长度 |
| 异常检测 | Isolation Forest | Autoencoder | 标签可用性 |
| 代码生成 | CodeLlama | DeepSeek-Coder | 私有化需求 |
| 图像生成 | Stable Diffusion | DALL-E 3 | 成本vs质量 |

### 8.2 部署架构选择

| 场景 | 推荐架构 | 关键组件 |
|-----|---------|---------|
| 高并发在线服务 | 微服务 + 推理服务网格 | Triton, KServe |
| 边缘部署 | 模型量化 + 边缘推理框架 | TensorRT, ONNX Runtime |
| 批处理 | 分布式训练 + 批推理 | Spark + Horovod |
| 实时流处理 | 流计算 + 在线学习 | Flink + 在线模型更新 |

## 9. 最佳实践总结

### 9.1 模型开发流程

```
1. 问题定义 → 2. 数据收集 → 3. 特征工程 → 4. 模型选择
      ↓            ↓            ↓            ↓
5. 训练调优 → 6. 评估验证 → 7. 部署上线 → 8. 监控迭代
```

### 9.2 关键成功因素

| 因素 | 最佳实践 |
|-----|---------|
| 数据质量 | 数据清洗、标注质量、数据增强 |
| 特征工程 | 领域知识、自动化特征、特征选择 |
| 模型选择 | 从简单开始、逐步复杂化 |
| 评估指标 | 业务指标优先、多维度评估 |
| 部署运维 | A/B测试、灰度发布、监控告警 |
| 团队协作 | MLOps流程、版本管理、文档化 |

### 9.3 常见陷阱与避免方法

| 陷阱 | 避免方法 |
|-----|---------|
| 数据泄露 | 严格分离训练/验证/测试集 |
| 过拟合 | 正则化、早停、交叉验证 |
| 评估偏差 | 使用业务相关指标 |
| 概念漂移 | 持续监控、在线学习 |
| 技术债务 | MLOps最佳实践 |
