## 一、思维导图格式规范

### 1.1 Mermaid思维导图格式

**基本格式**：

```markdown
```mermaid
mindmap
  root((根节点))
    分支1
      子分支1.1
      子分支1.2
    分支2
      子分支2.1
      子分支2.2
```

```

**示例**：
```markdown
```mermaid
mindmap
  root((AI三层模型架构))
    执行层
      图灵机抽象
      GPU矩阵运算
      CUDA优化
    控制层
      形式文法
      λ演算
      Prompt工程
    数据层
      概率论
      微分几何
      Transformer
```

```

### 1.2 思维导图元数据

**格式**：
```markdown
## X.X 思维导图名称

**主题**：主题描述

```mermaid
mindmap
  ...
```

**关联文档**：

- [文档1](./path1.md#章节)
- [文档2](./path2.md#章节)

```

### 1.3 思维导图编号规范

**编号格式**：`思维导图X`（按文档内顺序编号）

**示例**：
- 思维导图1：DKB架构全景
- 思维导图2：哲学谱系转译
- 思维导图3：IT技术哲学四重结构

## 三、证明树格式规范

### 3.1 Mermaid证明树格式

**基本格式**：

```markdown
```mermaid
graph TD
    A[前提1] --> C[结论]
    B[前提2] --> C
    C --> D[最终结论]
```

```

**示例**：
```markdown
```mermaid
graph TD
    A[公理A1] --> C[引理L1]
    B[公理A2] --> C
    C --> D[定理T1]
    E[公理A3] --> D
```

```

### 3.2 证明树元数据

**格式**：
```markdown
### X.X 证明树名称

3. 步骤3：...

**关联文档**：

```

### 3.3 证明树编号规范

## 四、决策树格式规范

### 4.1 Mermaid决策树格式

**基本格式**：
```markdown
```mermaid
graph TD
    A[决策点] -->|条件1| B[选项1]
    A -->|条件2| C[选项2]
    B --> D[结果1]
    C --> E[结果2]
```

```

**示例**：
```markdown
```mermaid
graph TD
    A[选择AI架构] -->|需要确定性| B[执行层]
    A -->|需要灵活性| C[控制层]
    A -->|需要泛化能力| D[数据层]
    B --> E[图灵计算模型]
    C --> F[形式语言模型]
    D --> G[数学概率模型]
```

```

## 七、统一格式要求

### 7.1 文档结构要求

所有思维表征文档应包含：
1. **标题**：思维表征方式名称
2. **元数据**：创建日期、最后更新、维护者、文档版本
3. **执行摘要**：文档内容概述
4. **主体内容**：思维表征方式的具体内容
5. **参考文档**：关联文档链接

### 7.2 编号规范

- **思维导图**：思维导图1, 思维导图2, ...
- **对比矩阵**：矩阵1, 矩阵2, ...
- **证明树**：证明树1, 证明树2, ...
- **决策树**：决策树1, 决策树2, ...
- **时间线**：时间线1, 时间线2, ...
- **层级模型**：层级模型1, 层级模型2, ...

### 7.3 引用规范

**格式**：`[思维表征名称](../文档路径.md#章节)`

**示例**：
- `[思维导图1](../Philosophy/model/02-思维导图总览.md#思维导图1)`
- `[矩阵1](../Philosophy/model/03-概念多维对比矩阵.md#矩阵1)`
- `[证明树1](../Philosophy/model/04-证明树图总览.md#证明树1)`
