## 一、收敛层级多维对比矩阵

| 层级 | 收敛度 C | 市场格局 | 技术特征 | 典型领域 | 驱动力 |
|------|----------|----------|----------|----------|--------|
| **L4 完全收敛** | C > 0.95 | 单一工业标准 | 架构/优化器统一 | 硬件层(CUDA)、数学层(Adam) | 技术+市场+标准 |
| **L3 准收敛** | 0.80 < C ≤ 0.95 | 1–2 方案事实垄断 | 2–3 架构并存，优者主导 | 架构层(Transformer)、能力层(基准) | 市场锁定+技术惯性 |
| **L2 收敛进行时** | 0.60 < C ≤ 0.80 | 双寡头竞争 | 多方案并存，收敛加速 | 应用层(MCP)、推理机制 | 资本效率+用户锁定 |
| **L1 未收敛** | 0 < C ≤ 0.60 | 百家争鸣 | 多方案并行，无主导 | Agent 协议、自我改进 | 技术探索 |
| **L0 反向分化** | C < 0 | 场景撕裂 | 方案分化加剧 | 垂直场景、专用架构 | 任务/硬件差异 |

## 二、Scaling Law 策略与收敛层级对应矩阵

| Scaling Law 策略 | 适用收敛层级 | 推理成本主导? | 典型应用 |
|------------------|--------------|---------------|----------|
| **Kaplan 原始** | L1–L2 | 否 | 早期训练阶段 |
| **Chinchilla (Hoffmann)** | L2–L3 | 否 | 训练阶段计算最优 |
| **Sardana 推理成本** | L3–L4 | 是 | 高 QPS、长上下文推理 |
| **Porian 三因素** | 复现/调和 | - | Kaplan vs Hoffmann 对齐 |
