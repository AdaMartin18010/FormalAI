# 反实践概念定义判断树

**创建日期**：2025-02-01  
**主题**：06-AI反实践判定系统  
**目的**：通过属性检验判定"给定实践 P 是否属于反实践"及"可判定性层级"

---

## 一、反实践归属判断树

```
给定实践 P(x)，判定是否属于反实践
│
├─ 1. 逻辑上 P(x) 是否可判定?（存在算法停机且输出正确）
│   ├─ 否 → [不可判定] 超出反实践判定范畴；参见哥德尔/图灵边界
│   └─ 是 ↓
│
├─ 2. 工程上 P(x) 是否可行?（可实现、可复现、可验证）
│   ├─ 是 → [规范实践] 非反实践
│   └─ 否 → [反实践] AntiPractice(P) = {x | LogicallyDecidable(P(x)) ∧ ¬EngineeringFeasible(P(x))}
│             └─ 逻辑可判定但工程不可行
```

---

## 二、反实践严重程度判定树

```
给定反实践实例，判定严重程度
│
├─ 影响范围小、可快速恢复?
│   └─ 是 → [轻度反实践] 06.3.2
│
├─ 影响中等、需人工介入修复?
│   └─ 是 → [中度反实践] 06.3.3
│
├─ 影响大、系统部分失效?
│   └─ 是 → [严重反实践] 06.3.4
│
└─ 系统崩溃、安全/合规重大风险?
    └─ 是 → [灾难性反实践] 06.3.5
```

## 三、三层模型反实践判定路径

```
给定 AI 系统，按层判定反实践
│
├─ 执行层反实践?
│   └─ 判定：可计算性违反、数值不稳定、硬件依赖不可移植
│   └─ 见 [06.2.2-执行层反实践判定](06.2.2-执行层反实践判定.md)
│
├─ 控制层反实践?
│   └─ 判定：形式语言滥用、Prompt 不可复现、安全协议形式化局限
│   └─ 见 [06.2.3-控制层反实践判定](06.2.3-控制层反实践判定.md)
│
├─ 数据层反实践?
│   └─ 判定：数据泄露、分布偏移未检测、奖励黑客
│   └─ 见 [06.2.4-数据层反实践判定](06.2.4-数据层反实践判定.md)
│
└─ 三层协同反实践?
    └─ 判定：层间契约违反、跨层不可判定性
    └─ 见 [06.2.5-三层协同反实践判定](06.2.5-三层协同反实践判定.md)
```

## 四、可判定性层级判定树

```
给定问题 Q，判定可判定性层级
│
├─ 存在算法在有限步内停机且输出正确?
│   ├─ 是 → [完全可判定] 06.4.2
│   └─ 否 ↓
│
├─ 存在半判定算法?（若答案为是则停机，若为否则可能不停机）
│   ├─ 是 → [半可判定] 06.4.3
│   └─ 否 → [不可判定] 06.4.4
│             └─ 停机问题归约；工程决策规则见 06.4.5
```
