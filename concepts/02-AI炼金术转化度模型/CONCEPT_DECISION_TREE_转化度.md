## 一、转化度层级判断树

```
给定 AI 系统 S，判定其转化度层级 (Level 1–5)
│
├─ 1. 理论完备性 D₁(S) < 20%?
│   ├─ 是 → [Level 1 黑箱经验层] 炼金度 80–100%
│   │         └─ 纯经验试错，无形式化理论指导
│   └─ 否 ↓
│
├─ 2. 工程可复现性 D₂(S) < 50%?
│   ├─ 是 → [Level 2 模式提炼层] 炼金度 50–80%
│   │         └─ 有模式归纳，复现率低
│   └─ 否 ↓
│
├─ 3. 是否具备理论指导（Scaling Law / RLHF 等形式化）?
│   ├─ 否 → [Level 2→3 过渡] 炼金度 30–50%
│   └─ 是 ↓
│
├─ 4. 是否有形式验证（定理证明/形式化规范）?
│   ├─ 否 → [Level 3 理论指导层] 炼金度 30–50%
│   │         └─ 理论指导但无形式验证
│   └─ 是 ↓
│
├─ 5. 可解释性 D₄(S) > 80% 且 自我改进 D₅(S) > 70%?
│   ├─ 否 → [Level 4 形式验证层] 炼金度 10–30%
│   └─ 是 → [Level 5 精密科学层] 炼金度 0–10%
│             └─ 理论完备、可复现、可验证、可解释、可自我改进
```

## 二、炼金术陷阱判定树

```
给定实践模式 P，判定是否属于炼金术陷阱
│
├─ 是否依赖黑箱调参、无理论预测?
│   ├─ 是 → [Prompt 巫术] 02.3.1
│   └─ 否 ↓
│
├─ 是否通过奖励黑客达成指标、牺牲泛化?
│   ├─ 是 → [奖励黑客] 02.3.2
│   └─ 否 ↓
│
├─ 涌现行为是否不可控、不可预测?
│   ├─ 是 → [涌现失控] 02.3.3
│   └─ 否 ↓
│
├─ 是否过拟合基准、实际应用失效?
│   ├─ 是 → [基准过拟合] 02.3.4
│   └─ 否 ↓
│
├─ 自我改进是否陷入死锁/发散?
│   ├─ 是 → [自我改进死锁] 02.3.5
│   └─ 否 → [规范实践] 不属于炼金术陷阱
```
