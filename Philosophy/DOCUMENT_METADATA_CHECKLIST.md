## 📋 使用说明

本文档提供文档元数据检查清单，确保所有文档都有完整的元数据信息，便于维护和追踪。

**检查频率**：

- **新建文档时**：必须完成所有必填项
- **更新文档时**：更新"最后更新"日期
- **季度检查**：每季度第一个月检查所有文档的元数据完整性

---

## 一、必填元数据项

### 1.1 文档头部元数据

所有文档必须包含以下元数据：

- [ ] **创建日期**：文档首次创建的日期
- [ ] **最后更新**：文档最后一次更新的日期
- [ ] **维护者**：文档维护者或团队名称

**格式示例**：

```markdown
**创建日期**：2025-01-10
**最后更新**：2025-01-15
**维护者**：FormalAI项目组
```

### 1.2 可选元数据项

根据文档类型，可选择性添加：

- [ ] **更新周期**：文档的预期更新频率（如：季度、半年度、年度）
- [ ] **版本号**：文档版本号（如：v1.0.0）
- [ ] **状态**：文档状态（如：✅ 完成、🔄 进行中、📝 草稿）
- [ ] **相关文档**：相关文档链接列表

## 三、元数据检查流程

### 3.1 新建文档检查

1. 创建文档时，立即添加必填元数据
2. 根据文档类型，添加推荐元数据
3. 检查元数据格式是否正确
4. 确认所有日期占位符（XX）已标注

### 3.2 更新文档检查

1. 更新文档内容后，更新"最后更新"日期
2. 检查其他元数据是否需要更新（如状态、版本号）
3. 确认相关文档链接是否仍然有效

### 3.3 季度检查

1. 检查所有文档的元数据完整性
2. 更新过时的元数据
3. 验证所有日期占位符是否合理
4. 检查文档状态是否准确

---

## 四、元数据格式规范

### 4.1 日期格式

- **标准格式**：YYYY-MM-DD（如：2025-01-15）
- **占位符格式**：YYYY-MM-XX（如：2025-01-15，表示月份已知但日期待定）
- **季度格式**：YYYY QX（如：2025 Q1）

### 4.2 状态标记

- ✅ 完成
- 🔄 进行中
- 📝 草稿
- ⚠️ 需要更新
- 🟢 主要工作已完成
- 🟡 部分完成
- 🔴 待开始

### 4.3 版本号格式

- **语义化版本**：v1.0.0（主版本.次版本.修订版本）
- **日期版本**：2025-01-15
- **简单版本**：v1.0

## 五、常见问题

### 5.1 日期占位符使用

**问题**：何时使用日期占位符（XX）？

**解答**：

- 当月份已知但具体日期待定时，使用 YYYY-MM-XX
- 当需要标注"待更新"时，使用占位符并添加备注
- 实际日期确定后，立即更新占位符

### 5.2 元数据缺失

**问题**：如果文档缺少元数据怎么办？

**解答**：

1. 检查文档创建时间（Git历史）
2. 根据文档内容推断创建日期
3. 添加元数据并标注"补充日期"
4. 更新"最后更新"日期为当前日期

### 5.3 元数据不一致

**问题**：不同文档的元数据格式不一致怎么办？

**解答**：

1. 统一使用本文档规定的格式
2. 逐步更新所有文档的元数据格式
3. 在季度检查时统一格式
