**目的**：通过属性检验判定"给定系统是否为/具备 Ontology 能力"

---

## 一、Ontology 类型判断树

```
给定系统 S，判定其 Ontology 类型
│
├─ 1. 是否有结构化概念/实体表示?
│   ├─ 否 → [无 Ontology] 纯向量/纯文本，无显式语义结构
│   └─ 是 ↓
│
├─ 2. 是否有显式关系/链接 (Links)?
│   ├─ 否 → [弱 Ontology] 仅有实体，无关系（如简单实体表）
│   └─ 是 ↓
│
├─ 3. 是否有业务语义（非通用 ID 匹配）?
│   ├─ 否 → [通用知识图谱] 如 DBpedia，业务无关
│   └─ 是 ↓
│
├─ 4. 是否有逻辑/规则层 (可执行)?
│   ├─ 否 → [静态 Ontology] 仅语义模型，无工具封装
│   └─ 是 ↓
│
├─ 5. 是否有决策历史/反馈闭环 (History)?
│   ├─ 否 → [半 DKB] 有 O+L，缺 H 层
│   └─ 是 → [完整 DKB Ontology] O+L+H 三元组
```

## 二、Ontology 能力成熟度判断树

```
给定 Ontology 系统，判定能力层级
│
├─ 数据整合能力
│   ├─ 无跨系统连接 → Level 0
│   ├─ 有 ETL/连接器，无语义对齐 → Level 1
│   ├─ 有语义对齐，无零 ETL 联邦查询 → Level 2
│   └─ 实时语义对齐 + 零 ETL 联邦查询 → Level 3
│
├─ 动态管理能力
│   ├─ 静态模型，无进化 → Level 0
│   ├─ 手工更新本体 → Level 1
│   ├─ 部分自动进化，无因果链维护 → Level 2
│   └─ 自适应本体进化 + 因果链维护 → Level 3
│
├─ 应用生成能力
│   ├─ 无 Agent/工具生成 → Level 0
│   ├─ 手工配置 Agent → Level 1
│   ├─ 领域本体驱动，部分精准 → Level 2
│   └─ 领域本体驱动的精准 Agent 生成 → Level 3
```

## 三、与 Stanford CS520 / W3C 的对齐

| 属性 | Stanford CS520 | W3C OWL | 项目 DKB-O |
|------|----------------|---------|------------|
| 数据模型 | 有向标注图 | RDF 三元组、OWL 类/属性 | 对象-链接-属性 |
| 推理 | 规则/图算法 | OWL 推理 | L 层逻辑封装 |
| 演化 | 知识图谱演化 | 本体版本 | H 层决策血缘 |
| 应用 | 检索、问答、推荐 | 语义 Web | 决策闭环 |
