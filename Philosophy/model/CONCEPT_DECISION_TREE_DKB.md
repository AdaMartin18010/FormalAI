# DKB 概念定义判断树

**创建日期**：2025-02-01  
**主题**：Philosophy - DKB (Decision Knowledge Base)  
**目的**：通过属性检验判定"给定系统是否为/具备 DKB 结构"

---

## 一、DKB 结构完备性判断树

```
给定系统 S，判定其 DKB 完备性
│
├─ 1. 是否有 O 层 (Ontology 语义内核)?
│   │   条件：对象(Objects) + 链接(Links) + 属性(Attributes) + 规则(Rules)
│   ├─ 否 → [非 DKB] 缺少语义内核
│   └─ 是 ↓
│
├─ 2. 是否有 L 层 (Logic 逻辑工具)?
│   │   条件：可执行函数（规则引擎、ML 模型、优化器、Agent 工具）
│   ├─ 否 → [残缺 DKB] 仅有 O，无逻辑封装
│   └─ 是 ↓
│
├─ 3. 是否有 H 层 (History 决策历史)?
│   │   条件：带血缘的决策日志，可反哺训练/本体
│   ├─ 否 → [半 DKB] O+L 但无闭环学习
│   └─ 是 → [完整 DKB] (O, L, H) 三元组
```

---

## 二、ARI 指数适用性判断树

```
判定 ARI = 语义对齐率 × log(逻辑封装度) × 行动闭环系数 是否可计算
│
├─ 语义对齐率可测量?（O 层与业务语义的对齐程度）
│   ├─ 否 → ARI 不适用，需先建立 O 层
│   └─ 是 ↓
│
├─ 逻辑封装度可测量?（L 层工具覆盖决策空间的比例）
│   ├─ 否 → ARI 不适用，需先建立 L 层
│   └─ 是 ↓
│
├─ 行动闭环系数可测量?（决策结果写回源系统的成功率）
│   ├─ 否 → ARI 不适用，需先建立 H 层与写回机制
│   └─ 是 → ARI 可计算；ARI ≥ 0.7 为生存阈值
```

## 三、公理-定理依赖判断（简化）

```

              │                      │
A2 决策闭环 ──┼──→ L2 逻辑封装必要性 ─┼──→ T1 DKB 基础设施定理
              │                      │
A3 知识复利 ──┼──→ L3 History 层价值 ─┤
              │                      │
A4 网络效应 ──┴──→ L4 边际成本递减 ──┘
```
