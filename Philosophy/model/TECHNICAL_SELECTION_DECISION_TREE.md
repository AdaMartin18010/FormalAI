## 一、形式化验证 vs 测试选型

```
是否需严格可验证性（安全关键/合规强制）?
│
├─ 是 → 形式化验证 (Coq/Lean) > 模型检测 > 定理证明
│       适用：航空、医疗、金融核心系统
│       权威：[FV-05] MIT Grove/Perennial
│
└─ 否 → 测试 > 静态分析 > 动态检测
        适用：一般企业应用、原型验证
```

---

## 二、Ontology vs RAG 选型

```
决策复杂度 > 5 系统联动?
│
├─ 否 → RAG/向量库 可能足够
│       条件：单域问答、文档检索、无闭环
│
└─ 是 → 需 Ontology / DKB
        条件：跨系统因果、决策血缘、闭环学习
        详见 model/05 决策树1、2
```

---

## 三、知识表示技术选型

```
业务语义是否需精确推理（非近似）?
│
├─ 否 → 向量相似性、Embedding 检索
│       成本低，HR 较高
│
└─ 是 → 结构化知识表示
        │
        ├─ 通用领域 → 知识图谱（DBpedia 等）
        │
        └─ 业务专属 → Ontology + DKB
                      需 O+L+H 三元组
```

---

## 四、AI 架构选型（与 model/03 矩阵2 对应）

```
ARI 目标?
│
├─ < 0.4 → 纯 LLM/RAG 或向量库（简单场景）
│
├─ 0.4–0.7 → 知识图谱（无 H 层闭环）
│
└─ ≥ 0.7 → DKB Ontology（完整 O+L+H）
           生存阈值，见 view02
```

研究目标?
