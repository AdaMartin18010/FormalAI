## 一、概念引用格式规范

### 1.1 基本引用格式

**格式**：`[概念名称](文档路径#章节锚点)`

**示例**：
- `[AI三层模型架构](../concepts/01-AI三层模型架构/README.md#四核心概念形式化定义)`
- `[DKB](../docs/03-formal-methods/03.5-DKB案例研究.md#dkb定义)`
- `[Ontology](../Philosophy/view01.md#ontology层)`

### 1.2 带形式化定义的引用

**格式**：`[概念名称](文档路径#章节锚点) - **形式化**：$公式$`

**示例**：
- `[执行层](../concepts/01-AI三层模型架构/README.md#41-执行层图灵计算模型形式化定义) - **形式化**：$E = (Q_E, \Sigma_E, \Gamma_E, \delta_E, q_{0_E}, B_E, F_E)$`
- `[Scaling Law](../concepts/03-Scaling Law与收敛分析/README.md#scaling-law定义) - **形式化**：$L(N) = a \cdot N^{-\alpha}$`

### 1.3 概念索引引用

**格式**：`[概念名称](../PROJECT_CONCEPT_SYSTEM.md#概念索引位置)`

**示例**：

## 三、形式化定义引用规范

### 3.1 形式化定义标记

**格式**：在概念定义后添加形式化表述

```markdown
**概念名称**：

- **定义**：概念的文字描述
- **形式化表述**：$数学公式$
- **来源**：`文档路径#章节`
```

**示例**：
```markdown
**执行层（图灵计算模型）**：

- **定义**：基于GPU矩阵运算的计算执行层
- **形式化表述**：$E = (Q_E, \Sigma_E, \Gamma_E, \delta_E, q_{0_E}, B_E, F_E)$
- **来源**：`concepts/01-AI三层模型架构/README.md#4.1`
```

### 3.2 形式化定义引用

**格式**：引用时包含形式化表述

```markdown
参见[执行层](../concepts/01-AI三层模型架构/README.md#41-执行层图灵计算模型形式化定义)：
$E = (Q_E, \Sigma_E, \Gamma_E, \delta_E, q_{0_E}, B_E, F_E)$
```

## 四、概念版本管理规范

### 4.1 概念版本标记

**格式**：在概念定义处标记版本

```markdown
**概念名称**（v2.0，2025-11-10更新）：

- **定义**：...
- **形式化表述**：...
- **变更历史**：
  - v1.0（2025-01-01）：初始定义
  - v2.0（2025-11-10）：添加形式化定义
```

### 4.2 概念版本引用

**格式**：引用时指定版本（可选）

```markdown
参见[执行层](../concepts/01-AI三层模型架构/README.md#执行层)（v2.0）
```

## 五、概念引用检查清单

### 5.1 引用前检查

- [ ] 确认概念在目标文档中存在
- [ ] 确认章节锚点正确
- [ ] 确认路径正确（相对路径或绝对路径）
- [ ] 确认概念定义是最新版本

### 5.2 引用后检查

- [ ] 链接可正常跳转
- [ ] 章节锚点准确
- [ ] 形式化定义格式正确
- [ ] 跨模块引用路径正确

### 5.3 定期审查

- [ ] 检查断链
- [ ] 检查概念定义更新
- [ ] 检查跨模块引用一致性
- [ ] 更新概念索引
