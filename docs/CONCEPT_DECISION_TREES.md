# 概念定义判断树集合 / Concept Decision Trees

**创建日期**：2025-02-02
**目的**：为FormalAI项目核心概念提供判断树，通过属性检验确定概念归属
**维护**：随概念定义更新而更新

## 一、使用说明

**判断树格式**：

- 节点：属性检验问题
- 分支：是/否答案
- 叶节点：概念归属或概念类型

**使用方法**：

1. 从根节点开始
2. 根据属性检验结果选择分支
3. 到达叶节点获得概念归属

## 二、意识概念判断树

### 2.1 意识(Consciousness)判断树

```
意识(Consciousness)?
├── 是否有主观体验(Qualia)?
│   ├── 否 → 非现象意识(可能为功能意识)
│   │   └── 是否有全局可访问性(Access)?
│   │       ├── 否 → 无意识
│   │       └── 是 → 功能意识(访问意识)
│   └── 是 ↓
├── 是否有全局可访问性(Access)?
│   ├── 否 → 最小意识(局部意识)
│   └── 是 ↓
├── 是否有自我指涉(Self-Reference)?
│   ├── 否 → 访问意识(Access Consciousness)
│   └── 是 → 自我反思意识(Self-Reflective Consciousness)
```

**权威依据**：

- **Qualia**: Chalmers (1995) "难问题" [CO-01]
- **Access Consciousness**: Block (1995) 访问意识 vs 现象意识
- **Self-Reference**: 自我模型理论

**应用场景**：

- 评估AI系统的意识程度
- 区分不同类型的意识
- 设计意识测试实验

## 三、Scaling Law判断树

### 3.1 Scaling Law类型判断树

```
Scaling Law类型?
├── 是否考虑推理成本?
│   ├── 是 → 推理成本扩展Scaling Law (Sardana 2024)
│   │   └── 是否考虑多变量?
│   │       ├── 是 → 多变量Scaling Law (N, D, C)
│   │       └── 否 → 单变量推理成本Scaling Law
│   └── 否 ↓
├── 是否考虑数据规模?
│   ├── 是 → Chinchilla Scaling Law (Hoffmann 2022)
│   │   └── D_opt ∝ N^0.74, N_opt ∝ D^1.35
│   └── 否 → Kaplan Scaling Law (2020)
│       └── L(N) ∝ N^(-α)
```

**权威依据**：

- **Kaplan**: Kaplan et al. (2020) [DL-06]
- **Chinchilla**: Hoffmann et al. (2022) [SL-01]
- **推理成本**: Sardana et al. (2024) [SL-02]

## 四、形式化验证方法判断树

### 4.1 形式化验证方法选择树

```
是否需要严格可验证性?
├── 是 ↓
│   ├── 是否需要完整状态空间验证?
│   │   ├── 是 → 模型检测(Model Checking)
│   │   │   └── 是否支持运行时验证?
│   │   │       ├── 是 → 运行时监控框架 (2025)
│   │   │       └── 否 → 静态模型检测器
│   │   └── 否 → 定理证明(Theorem Proving)
│   │       └── 是否涉及神经网络?
│   │           ├── 是 → 神经符号证明 (ProofNet++ 2025)
│   │           └── 否 → 传统定理证明器 (Coq/Lean)
│   └── 否 → 测试 > 静态分析
```

**权威依据**：

- **模型检测**: Clarke et al. (2018) [FV-01]
- **定理证明**: Hoare (1969) [FV-04]
- **ProofNet++**: arXiv 2505.24230 (2025) [FV-07]
- **运行时验证**: arXiv 2507.11987 (2025) [FV-10]

## 五、PAC 学习判断树

### 5.1 PAC 可学习性判断树

```
概念类是否 PAC 可学习?
├── 假设类 H 是否有限?
│   ├── 是 → 样本复杂度 m = O( (1/ε²)(log|H| + log(1/δ)) )
│   │   └── 使用 Hoeffding 界 [SLT-02][SLT-03]
│   └── 否 ↓
├── 是否有有限 VC 维 d?
│   ├── 是 → PAC 可学习，m = O( (d/ε²)(log(1/ε) + log(1/δ)) )
│   │   └── 使用 VC 维泛化界
│   └── 否 ↓
├── 是否可计算 Rademacher 复杂度?
│   ├── 是 → 使用 Rademacher 泛化界 [SLT-03]
│   └── 否 → 需进一步检验（如稳定性、PAC-Bayes）
```

**权威依据**：

- **PAC 定义**: Valiant (1984); Mohri et al. (2018) [SLT-02][SLT-03]
- **有限假设类**: 样本复杂度 O(1/ε²(log|H|+log(1/δ)))
- **VC 维**: [02.1.2 VC维理论](../02-machine-learning/02.1-统计学习理论/02.1.2-VC维理论.md)
- **Rademacher**: [CONCEPT_DEFINITION_SOURCE_TABLE_FIRST_BATCH](CONCEPT_DEFINITION_SOURCE_TABLE_FIRST_BATCH.md) §一

**应用场景**：

- 判断给定假设类是否 PAC 可学习
- 估计样本复杂度
- 与 [02.1.1 PAC学习理论](../02-machine-learning/02.1-统计学习理论/02.1.1-PAC学习理论.md) 对照

## 六、AI架构选择判断树

### 6.1 三层模型 vs DKB判断树

```
选择AI架构?
├── 是否需要企业级决策支持?
│   ├── 是 → DKB架构 (O, L, H三元组)
│   │   └── 是否需要Ontology语义层?
│   │       ├── 是 → DKB完整架构
│   │       └── 否 → 简化DKB (仅L, H层)
│   └── 否 ↓
├── 是否需要形式化控制?
│   ├── 是 → 三层模型 (执行/控制/数据)
│   │   └── 是否需要概率建模?
│   │       ├── 是 → 完整三层模型
│   │       └── 否 → 两层模型 (执行/控制)
│   └── 否 → 单一架构 (根据具体需求)
```

**权威依据**：

- **三层模型**: concepts/01-AI三层模型架构
- **DKB**: Philosophy/view02, model/01

## 七、意识理论选择判断树

### 7.1 意识理论适用性判断树

```
选择意识理论?
├── 是否需要可计算性?
│   ├── 是 ↓
│   │   ├── 是否需要高可检验性?
│   │   │   ├── 是 → GWT (全局工作空间理论)
│   │   │   │   └── 是否需要质量控制?
│   │   │   │       ├── 是 → GWT+HOT互补理论 (2025)
│   │   │   │       └── 否 → 纯GWT理论
│   │   │   └── 否 → HOT (高阶思维理论)
│   │   └── 否 → IIT (整合信息理论)

```

**权威依据**：

## 八、收敛层级判断树

### 8.1 收敛层级判定树

```
判定收敛层级?
├── 收敛度C > 0.95?
│   ├── 是 → L4 (完全收敛，工业标准)
│   └── 否 ↓
├── 收敛度C > 0.80?
│   ├── 是 → L3 (准收敛，事实垄断)
│   └── 否 ↓
├── 收敛度C > 0.60?
│   ├── 是 → L2 (收敛进行时，双寡头竞争)
│   └── 否 ↓
├── 收敛度C > 0?
│   ├── 是 → L1 (未收敛，百家争鸣)
│   └── 否 → L0 (反向分化，场景撕裂)
```

**收敛度计算公式**：
$$C_i = \frac{\text{采用标准方案的产品数}}{\text{总产品数}} \times \frac{\text{标准方案性能}}{\text{最佳方案性能}}$$

**权威依据**：

- concepts/03-Scaling Law与收敛分析
