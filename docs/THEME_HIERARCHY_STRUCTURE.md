## 三、文档组织规范

### 3.1 目录命名规范

- **格式**：`{序号}-{主题名称}`
- **示例**：`01-foundations`, `02-machine-learning`
- **规则**：
  - 序号使用两位数字，用点号分隔层次
  - 主题名称使用连字符分隔
  - 支持中英文主题名称

### 3.2 文件命名规范

- **主文档**：`README.md`（每个主题目录必须包含）
- **子文档**：`{序号}.{子序号}-{子主题名称}.md`
- **示例**：`02.1.1-PAC学习理论.md`

### 3.3 内容组织规范

每个README.md必须包含：

1. **文档头部**
   - 标题（中英德法四语言）
   - 返回全局导航链接
   - 学习路径链接

2. **核心内容**
   - 概述
   - 核心概念定义
   - 理论框架
   - 2025年最新发展章节
   - 代码示例（如适用）

3. **导航链接**
   - 前置依赖
   - 后续应用
   - 相关章节
   - 同级主题

4. **文档尾部**
   - 参考文献
   - 进一步阅读
   - 最后更新日期
