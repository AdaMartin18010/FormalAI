**创建日期**：2025-02-02
**目的**：为FormalAI项目提供技术选型决策树，帮助选择合适的技术方案
**维护**：随技术发展更新

## 一、使用说明

**决策树格式**：

- 节点：决策条件或问题
- 分支：条件满足/不满足
- 叶节点：推荐的技术方案或工具

**使用方法**：

1. 从根节点开始
2. 根据实际情况回答条件问题
3. 沿着分支到达推荐方案

## 二、形式化验证方法选型决策树

### 2.1 形式化验证方法选择

```
是否需要严格可验证性?
├── 是 ↓
│   ├── 是否需要完整状态空间验证?
│   │   ├── 是 → 模型检测(Model Checking)
│   │   │   └── 系统规模?
│   │   │       ├── 小规模(<10^6状态) → 显式模型检测器(SPIN, NuSMV)
│   │   │       ├── 中规模(10^6-10^9) → 符号模型检测器(CBMC, BDD-based)
│   │   │       └── 大规模(>10^9) → 抽象解释 + 模型检测混合
│   │   └── 否 → 定理证明(Theorem Proving)
│   │       └── 是否涉及神经网络?
│   │           ├── 是 → 神经符号证明 (ProofNet++ 2025)
│   │           │   └── 是否需要自纠正?
│   │           │       ├── 是 → ProofNet++ with self-correction
│   │           │       └── 否 → 传统神经符号验证
│   │           └── 否 → 传统定理证明器
│   │               └── 证明复杂度?
│   │                   ├── 低 → 自动化证明器(Z3, CVC4)
│   │                   ├── 中 → 交互式证明器(Coq, Lean)
│   │                   └── 高 → 专家级证明器(Isabelle/HOL)
│   └── 否 → 测试 > 静态分析
│       └── 是否需要运行时保证?
│           ├── 是 → 运行时监控框架 (2025动态验证)
│           └── 否 → 静态分析工具
```

**权威依据**：

- **模型检测**: Clarke et al. (2018) [FV-01]
- **定理证明**: Hoare (1969) [FV-04]
- **ProofNet++**: arXiv 2505.24230 (2025) [FV-07]
- **运行时验证**: arXiv 2507.11987 (2025) [FV-10]

## 三、神经网络验证工具选型决策树

### 3.1 神经网络验证工具选择

```
需要验证的神经网络类型?
├── ReLU网络 → α,β-CROWN框架
│   └── 是否需要分支定界?
│       ├── 是 → GenBaB (2025扩展版)
│       └── 否 → CROWN基础版
├── 非线性激活网络 → GenBaB (2025)
│   └── 激活函数类型?
│       ├── Sigmoid/Tanh/Sine/GeLU → GenBaB支持
│       └── 其他 → Marabou 2.0 (2025综合验证器)
└── 大规模网络 → Marabou 2.0
    └── 是否需要完整验证?
        ├── 是 → Marabou 2.0完整验证
        └── 否 → 抽象解释 + 部分验证
```

**权威依据**：

- **GenBaB**: [FV-08] 扩展神经网络验证到非线性激活
- **Marabou 2.0**: [FV-09] 综合神经网络形式化分析器

## 四、AI架构选型决策树

### 4.1 三层模型 vs DKB vs 其他架构

```
应用场景类型?
├── 企业决策支持系统 → DKB架构
│   └── 是否需要Ontology语义层?
│       ├── 是 → DKB完整架构 (O, L, H)
│       │   └── 是否需要历史决策追踪?
│       │       ├── 是 → DKB完整架构
│       │       └── 否 → 简化DKB (O, L)
│       └── 否 → 三层模型
├── AI系统架构分析 → 三层模型
│   └── 是否需要形式化控制?
│       ├── 是 → 三层模型 (执行/控制/数据)
│       └── 否 → 单一架构
└── 通用AI系统 → 根据具体需求选择
    └── 是否需要理论分析?
        ├── 是 → 三层模型
        └── 否 → 实用架构(Transformer等)
```

**权威依据**：

- **三层模型**: concepts/01-AI三层模型架构
- **DKB**: Philosophy/view02, model/01

## 五、意识理论选型决策树

### 5.1 意识理论选择（用于AI系统设计）

```
设计目标?
├── 需要可计算性 → GWT或HOT
│   └── 是否需要广播机制?
│       ├── 是 → GWT (全局工作空间理论)
│       │   └── 是否需要质量控制?
│       │       ├── 是 → GWT+HOT互补理论 (2025)
│       │       └── 否 → 纯GWT
│       └── 否 → HOT (高阶思维理论)
├── 需要理论完整性 → IIT
│   └── Φ计算复杂度可接受?
│       ├── 是 → IIT (整合信息理论)
│       └── 否 → 简化IIT或GWT
└── 需要最新理论 → RCUET或MCT
    └── 是否需要模块化?
        ├── 是 → MCT (模块化意识理论 2025)
        └── 否 → RCUET (2025)
```

**权威依据**：

- **IIT**: Tononi et al. (2016) [CO-02]
- **GWT**: Dehaene (2014) [CO-03]
- **GWT+HOT**: Phua et al. (2025) [CO-05]
- **RCUET**: arXiv 2505.01464 (2025) [CO-06]

## 六、Scaling Law应用选型决策树

### 6.1 Scaling Law模型选择

```
应用场景?
├── 训练阶段规划 → Chinchilla Scaling Law
│   └── 是否考虑推理成本?
│       ├── 是 → Sardana推理成本扩展 (2024)
│       └── 否 → Hoffmann Chinchilla (2022)
├── 推理阶段优化 → Sardana推理成本扩展
│   └── 固定推理预算?
│       ├── 是 → Sardana扩展模型
│       └── 否 → Chinchilla基础模型
└── 理论分析 → Kaplan原始模型
    └── 是否需要多变量?
        ├── 是 → 多变量扩展模型
        └── 否 → Kaplan单变量模型
```

**权威依据**：

- **Kaplan**: Kaplan et al. (2020) [DL-06]
- **Chinchilla**: Hoffmann et al. (2022) [SL-01]
- **推理成本**: Sardana et al. (2024) [SL-02]
